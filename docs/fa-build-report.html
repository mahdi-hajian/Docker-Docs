خروجی گرفتن از ONLYOFFICE Docker-Docs — گزارش کار

این صفحه خلاصه می‌کند از ریپوی Docker-Docs چطور ایمیج ساختیم، چه مشکلاتی پیش آمد، و چه تغییراتی اعمال شد تا stack محلی بالا بیاید.

هدف

  • ساخت ایمیج‌های Docs برای Docker Compose و بعداً Kubernetes
  • اجرای non-root (کاربر ds، UID/GID 101)
  • رفع خطاهای بیلد/اجرا روی ویندوز
  • فهم تفاوت Community / Developer (-de) و لایسنس

این ریپو چه می‌کند؟

ایمیج رسمی یک‌تکه onlyoffice/documentserver را patch نمی‌کند. جریان بیلد:

  1. پایه: fedora
  2. دانلود و نصب RPM فقط‌آفیس
  3. بسته‌بندی برای cluster / سرویس‌های جدا
  4. اجرا با USER ds (non-root)

کارهایی که انجام شد

۱) بیلد روی ویندوز

  • افزودن build.ps1 (معادل build.sh)
  • Git Bash: ./build.sh
  • PowerShell: .\build.ps1

۲) مشکل CRLF (bash\r)

علائم: Restart مداوم با پیام env: 'bash\r': No such file or directory

علت: CRLF ویندوز داخل entrypointهای لینوکسی

اقدامات: تبدیل به LF، فایل .gitattributes، و در Dockerfile:

RUN sed -i 's/\r$//' ...entrypoint... && chmod +x ...

اگر ایمیج خراب را push کنی، روی Kubernetes لینوکس هم همان خطا را می‌گیری. CRLF مشکل زمان بیلد است نه فقط اجرا روی ویندوز.

۳) مشکل docker compose

علائم: نبودن image/build برای postgresql و metrics

اقدامات روی compose:

  • افزودن image برای metrics و postgresql
  • اختیاری کردن example با profile
  • اصلاح healthcheck به -U myuser -d mydb
  • تنظیم EXAMPLE_HOST_PORT=docservice:8000 تا nginx بدون example بالا بماند

۴) نتیجه موفق

  • healthcheck = true
  • proxy روی پورت 80 healthy
  • داخل کانتینر: uid=101(ds)
  • هشدار نبودن license.lic برای -de طبیعی است
