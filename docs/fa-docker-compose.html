راهنمای استفاده با Docker Compose

مراحل بیلد و اجرای ONLYOFFICE Docs از ریپوی Docker-Docs روی سیستم محلی (ویندوز/لینوکس).

پیش‌نیاز

  • Docker Desktop / Docker Engine
  • Docker Compose v2
  • حداقل چند گیگابایت RAM آزاد (ترجیحاً ۴GB+ فقط برای Docs)
  • فایل .env در ریشه پروژه

تنظیم .env

نمونه برای Developer:

ACCOUNT_NAME=onlyoffice
PREFIX_NAME=docs
PRODUCT_EDITION=-de
RELEASE_VERSION=-9.4.1
DOCKER_TAG=latest
JWT_ENABLED=true
JWT_HEADER=Authorization
JWT_SECRET=secret

برای Community مقدار PRODUCT_EDITION را خالی بگذار.

گام ۱ — بیلد ایمیج‌ها

PowerShell:

cd E:\github\Docker-Docs
.\build.ps1

Git Bash:

cd /e/github/Docker-Docs
./build.sh

خروجی مهم: onlyoffice/docs-cluster-de:latest و همچنین utils / metrics / postgresql.

گام ۲ — اجرا

docker compose up -d
docker compose ps
curl http://localhost/healthcheck

اگر همه چیز درست باشد، خروجی healthcheck باید true باشد و صفحه در http://localhost باز شود.

دستورات مفید

# وضعیت
docker compose ps

# لاگ
docker compose logs -f proxy docservice converter

# چک non-root
docker compose exec docservice id

# scale converter
docker compose up -d --scale converter=3

# توقف
docker compose down

فونت کاستوم

  1. فایل‌های .ttf را در پوشه fonts/ بگذار
  2. دوباره بیلد کن (build.ps1 / build.sh)
  3. stack را بالا بیاور

عیب‌یابی رایج

خطاکار
bash\r اسکریپت‌ها LF باشند / Dockerfile کار strip CRLF را داشته باشد و دوباره بیلد کن
neither an image nor a build برای postgresql و metrics فیلد image تعریف شده باشد
proxy روی upstream example می‌میرد EXAMPLE_HOST_PORT را به هاست موجود بده یا example را بالا بیاور
license.lic ENOENT برای -de بدون لایسنس طبیعی است؛ سرویس می‌تواند بالا باشد ولی کامل/cluster تجاری محدود است

نکته ایمیج‌های unused

ایمیج‌هایی مثل docs-proxy-de / docs-docservice-de / docs-converter-de ممکن است ساخته شوند ولی در compose استفاده نشوند، چون همه نقش‌ها از docs-cluster-de می‌آیند و نقش با command/entrypoint عوض می‌شود. این به معنی «روی Kubernetes حتماً لازم‌اند» نیست؛ در Helm هم می‌توان همه را به cluster برد. توضیح تخصصی کامل: fa-images-cluster-vs-split.html
