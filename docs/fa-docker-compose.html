Docker Compose — چه چیزی بالا می‌آید؟ اجباری یا اختیاری؟

این صفحه فقط برای اجرای لوکال با docker compose است. هدف: بدانی هر سرویس به چه دردی می‌خورد، کدام بدون آن Docs کار نمی‌کند، و کدام را می‌شود حذف کرد.

لیست ایمیج‌های رجیستری تو (نمونه): 
repo.mohaymen.ir:3060/starrelease/onlyoffice/docs-cluster-de:none-root
repo.mohaymen.ir:3060/starrelease/onlyoffice/docs-proxy-de:none-root
repo.mohaymen.ir:3060/starrelease/onlyoffice/docs-docservice-de:none-root
repo.mohaymen.ir:3060/starrelease/onlyoffice/docs-converter-de:none-root
repo.mohaymen.ir:3060/starrelease/onlyoffice/docs-utils:none-root
repo.mohaymen.ir:3060/starrelease/onlyoffice/docs-metrics:none-root
repo.mohaymen.ir:3060/starrelease/onlyoffice/docs-postgresql:none-root

۱) فرق مهم: ایمیج ≠ سرویس/کانتینر

  • ایمیج = فایل روی رجیستری (مثل docs-cluster-de)
  • سرویس Compose = کانتینر در حال اجرا با یک نقش مشخص (مثل docservice)

در Compose این ریپو، سرویس‌های Docs معمولاً همه از یک ایمیج (docs-cluster-de) بالا می‌آیند و فقط command/entrypoint فرق می‌کند.

۲) سرویس‌هایی که Compose بالا می‌آورد — اجباری / اختیاری

الف) هسته Docs — بدون این‌ها ادیتور کار نمی‌کند

سرویس Compose وضعیت به چه درد می‌خورد؟ ایمیج پیشنهادی تو اگر نباشد چه می‌شود؟
proxy اجباری ورودی وب (پورت ۸۰). Nginx جلوی بقیه سرویس‌ها می‌ایستد و درخواست را پخش می‌کند. docs-cluster-de:none-root
(با entrypoint پروکسی)
یا docs-proxy-de:none-root		 از بیرون به Docs دسترسی نداری.
docservice اجباری قلب سیستم: باز کردن سند، API، همکاری هم‌زمان، وب‌سوکت ادیتور. docs-cluster-de:none-root
+ command: [docservice]
یا docs-docservice-de:none-root		 ادیتور بالا نمی‌آید / healthcheck خراب است.
converter اجباری برای کار واقعی تبدیل فرمت فایل (DOCX↔PDF و …). کارهای سنگین CPU. docs-cluster-de:none-root
+ command: [converter]
یا docs-converter-de:none-root		 باز کردن بعضی فرمت‌ها / تبدیل / ذخیرهٔ برخی عملیات می‌شکند.

ب) زیرساخت داده — در Compose داخل خود فایل آمده (برای راحتی لوکال)

سرویس Compose وضعیت در این ریپو به چه درد می‌خورد؟ ایمیج اگر نباشد؟
postgresql اجباری ذخیرهٔ وضعیت اسناد/سشن‌های سمت سرور. باید schema فقط‌آفیس را داشته باشد. docs-postgresql:none-root
یا Postgres معمولی + createdb.sql		 docservice بالا نمی‌آید یا مدام خطا می‌دهد.
redis اجباری کش و هماهنگی وضعیت بین سرویس‌ها. redis:7 (رسمی، از Docker Hub) سرویس‌ها fail می‌شوند.
rabbitmq اجباری صف پیام بین docservice و converter. rabbitmq:3 (رسمی) تبدیل/هماهنگی converter خراب می‌شود.

نکته: در Compose این سه تا «اجباری»‌اند چون فایل compose خودش آن‌ها را تعریف کرده و docservice بهشان depends_on دارد. یعنی برای اجرای همین فایل، باید باشند. در Kubernetes معمولاً این سه را جدا (خارج از chart Docs یا به‌عنوان dependency) می‌آوری — نه لزوماً با ایمیج‌های ONLYOFFICE.

ج) اختیاری در Compose

سرویس وضعیت به چه درد می‌خورد؟ ایمیج اگر نباشد؟
adminpanel اختیاری برای ادیتور پنل مدیریت تنظیمات (مفید برای admin، لازم برای باز شدن سند نیست). docs-cluster-de:none-root + command: [adminpanel] سند باز می‌شود؛ پنل ادمین نداری.
metrics اختیاری جمع‌آوری متریک StatsD. در compose فعلی روشن است چون env روی docservice به آن اشاره می‌کند. docs-metrics:none-root اگر METRICS_ENABLED=true باشد و سرویس نباشد، بهتر است metrics را نگه داری یا متریک را خاموش کنی.
utils اختیاری (لوکال تقریباً بی‌استفاده) اسکریپت‌های کمکی بیشتر برای محیط Kubernetes/observer. docs-utils:none-root برای تست ادیتور لوکال معمولاً لازم نیست.
example اختیاری اپ نمونه برای دموی باز کردن سند. docs-example (اگر بیلد شده باشد) Docs خودش کار می‌کند؛ فقط صفحهٔ نمونه نداری. در فایل فعلی با profile جدا شده.

۳) حداقل مجموعه برای «Docs کار کند» در Compose

اجباری واقعی:

  1. proxy
  2. docservice
  3. converter (حداقل ۱)
  4. postgresql (با schema)
  5. redis
  6. rabbitmq

اختیاری:

  • adminpanel
  • metrics (اگر متریک خاموش باشد)
  • utils
  • example

۴) ایمیج‌هایی که ساختی ولی Compose لزوماً استفاده نمی‌کند

ایمیجآیا Compose این ریپو به آن نیاز دارد؟توضیح
docs-cluster-de بله — اصلی پایهٔ proxy/docservice/converter/adminpanel در compose فعلی.
docs-proxy-de نه (اختیاری) همان محتوای cluster با entrypoint پیش‌فرض proxy. فقط اگر بخواهی به‌جای cluster از این تگ استفاده کنی.
docs-docservice-de نه (اختیاری) مستعار نقش docservice.
docs-converter-de نه (اختیاری) مستعار نقش converter.
docs-postgresql بله در compose فعلی یا جایگزینش Postgres معمولی + SQL.
docs-metrics بستگی به METRICS_ENABLED در فایل فعلی معمولاً بالا می‌آید.
docs-utils نه برای ادیتور در compose تعریف شده ولی برای تست سند لازم نیست.

۵) اجرا

docker compose up -d
docker compose ps
curl http://localhost/healthcheck
# انتظار: true

پورت: http://localhost (نگاشت 80:8888)

۶) مقایسه با Kubernetes

در Compose، Postgres/Redis/RabbitMQ هم داخل همان فایل «اجباری» شده‌اند چون هدف فایل، یک stack کامل لوکال است. در Kubernetes معمولاً فقط Deploymentهای Docs اجباری‌اند و DB/Redis/MQ را جدا مدیریت می‌کنی. جزئیات: fa-kubernetes-helm.html
